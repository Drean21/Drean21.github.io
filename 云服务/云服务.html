域名选购

永久免费的二级域名：eu.org，支持托管到CloudFlare

该域名非常容易被阻断

在很早之前这个域名已经被大陆阻断，但还是能够通过CloudFlare代理进行访问
但是，现如今该域名的tls已被阻断，导致我的博客和项目无法正常访问，正在更换为小姨子xyz域名

域名的选择：

  • 没得选的情况下，可以加前缀，比如想要rgzn，可以改为rgznapp；
  • 不要有-这样的；
  • 好的前缀可能比后缀更为重要
  • 尽量不要用top域名，虽然编译但服务在国内，容易被查水表

可以从：Cheap Domain Names & Web Hosting Starting at $0.99! | NameSilo选购域名，比较老牌，相对不错，支持支付宝

目前比较便宜的就是6位纯数字域名，用一些有特殊意义的数字做域名其实也还不错，但这个网站的DNS和其他服务做的相当简陋，我们肯定还是要选择把域名托管到CloudFlare去管理

一般都是在你购买的域名管理商中将名称服务器换成你CloudFlare的名称服务器即可
如何确保你域名的名称服务器更改成功了呢？可以通过如下网址查询：NS Lookup - Lookup Nameserver of any Domain (dnschecker.org)

太可恶了，突然之间，我使用Vercel和CloudFlare反代的博客，也就是本站就国内失联了！！！
先换用GithubPage吧，上线的项目隧道也只有http能用了现在

可以用来免费部署任意服务，免费用户物理关机，严禁滥用
Replit容器，但不管怎么样，它的自由度相对来说会挺高挺像一个Linux主机的

其他容器，基本上是拉取Git，然后部署静态服务，不像Replit那么自由

下面总结一些可以用来部署静态服务托管的途径

  • CloudFlare：https://dash.cloudflare.com
  • GitHub Pages：https://pages.github.com/
  • Vercel：https://vercel.com/new
    我其实用不多，基本上都是在使用CloudFlare（面向国内推荐使用Vercel，CloudFlare只做DNS解析）

CloudFlare 提供每日10000次请求额度的worker无服务器应用、workerPage静态页面部署服务，可以用来搭建反向代理服务利用它家CDN实现无服务器的简单网页和服务，但是不建议使用它来搭建面向国内的博客，实测加载比较慢尤其没有缓存的用户第一次访问（动画元素加载慢）

但搭建国内博客建议使用：Vercel，但必须使用自定义域否则国内无法访问（疑似被墙）

  • 使用CloudFlare完成域名的DNS解析（不代理其流量）

  • 这时候国内访问起飞

  • vercel还支持：

    • 设置服务器器运行地区，可以选择亚洲：

    • 重定向流量，官方文档，因为vercel免费100G/月，而CloudFlare的流量你基本可以无视，可以将rss订阅等内容转发到他家服务上去。简单例子，在项目更目录下创建vercel.json文件，如下：

        {
	"regions": [
		"hkg1"
	],
	"redirects": [{
		"source": "/lib/rss.xml",
		"destination": "https://blog.cytl2.eu.org/lib/rss.xml"
	}]
 }

    • 设置缓存时间、自定义404页面等，参见上面官方文档

图床

目前最稳定的方案无过于借助B站API，
xlzy520/typora-plugin-bilibili: Typora粘贴图片自动上传到Bilibili图床，也可以自定义修改成任意其他图床接口。使用教程：https://b23.tv/urxCc3 (github.com)
但借助别人的存储一般会做防盗链等限制，为了保证自家网站能正常加载显示，我们需要在html文档的head标签中加入，语雀也是同理

<meta name="referrer" content="no-referrer" />

利用CloudFlare+Telegraph这样的API也可以搭建一个稳定的图床

使用Telegraph零成本构建免费的图像托管/无审查/可以上传任何文件，不仅仅是图像文件 - 云线路
当然，最稳定的肯定是使用对象存储，比如七牛云
